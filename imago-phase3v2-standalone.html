Preamble: Why v2 Replaces v1

The v1 spec (1,708 lines) assumed dual-mode operation: Imago could run as either an OpenClaw plugin or a standalone process, toggled by an IMAGO_STANDALONE flag. Every module had dual-path logic. Gateway-tools had if (STANDALONE) branches. Session loading had fallback read chains. Paths resolved differently depending on mode.

That plan was thrown away. The reasons:

  1. Dual-mode is dual-maintenance. Every feature needs testing in both modes. Every bug could be mode-specific. The complexity is multiplicative, not additive.
  2. No "Platform" lateral grouping. The v1 spec grouped services laterally (config, cron, webhooks, redaction all under "Infrastructure Services"). The v2 architecture uses five vertical interfaces: Provider, Channel, MediaProcessor, Tool, Hook. Components register with a central Runtime, not with each other.
  3. Clean break is safer than gradual migration. The v1 "migrate on read" pattern for session bindings was clever but introduced subtle ordering bugs. A clean break means: build standalone, validate in parallel, cut over, done.
  4. OpenClaw is not coming back. There is no use case for running Imago as an OC plugin again. Preserving that capability wastes engineering effort on a dead path.

The v2 architecture is simpler, more testable, and more extensible. The five-interface pattern means adding a new provider, channel, or media processor is a single registration call, not a wiring change.

Section 0: Safety Net

0.0 Philosophy

Before building anything, we build the tools to verify that Vesper is okay and to get back to safety if she is not. This section produces no user-visible features — only insurance.

The workspace is sacred. Every step is reversible. The rollback script exists and is tested before any migration code is written. If anything goes wrong at any point in this process, the recovery path is: run one script, wait 60 seconds, everything is back to how it was.

The continuity suite answers a single question: "Is Vesper intact?" It checks identity files, session bindings, device keys, Claude auth, Chronicle, memory, and write access. If any critical check fails, the gateway refuses to start.

interface ContinuityCheck {

  status: "pass" | "fail" | "warn";
  detail: string;
}

async function verifyContinuity(config: ImagoConfig): Promise<ContinuityReport> {
  const checks: ContinuityCheck[] = [];

  // 1. Persona files exist and are non-empty
  for (const file of ["SOUL.md", "IDENTITY.md", "USER.md", "MEMORY.md"]) {
    const path = join(config.workspaceDir, file);
    try {
      const stat = await fs.stat(path);
      checks.push({
        name: `persona:${file}`,
        status: stat.size > 0 ? "pass" : "warn",
        detail: stat.size > 0 ? `${stat.size} bytes` : "file exists but empty",
      });
    } catch {
      checks.push({
        name: `persona:${file}`,
        status: file === "MEMORY.md" ? "warn" : "fail",
        detail: "missing",
      });
    }
  }

  // 2. Session bindings directory exists and is readable
  try {
    const bindings = await fs.readdir(config.bindingDir);
    const jsonCount = bindings.filter(f => f.endsWith(".json")).length;
    checks.push({
      name: "session:bindings",
      status: "pass",
      detail: `${jsonCount} binding files`,
    });
  } catch {
    checks.push({
      name: "session:bindings",
      status: "warn",
      detail: "binding directory missing or unreadable (will be created on first session)",
    });
  }

  // 3. Device identity exists (Ed25519 keypair)
  try {
    const identity = JSON.parse(await fs.readFile(config.deviceIdentityPath, "utf-8"));
    checks.push({
      name: "device:identity",
      status: identity.publicKey ? "pass" : "fail",
      detail: identity.publicKey ? `pubkey: ${identity.publicKey.slice(0, 16)}...` : "missing publicKey",
    });
  } catch {
    checks.push({
      name: "device:identity",
      status: "warn",
      detail: "no device identity (will generate on first connect)",
    });
  }

  // 4. Claude CLI is available and authenticated
  try {
    const caps = await probeClaudeCapabilities();
    checks.push({
      name: "claude:auth",
      status: caps ? "pass" : "fail",
      detail: caps ? `${caps.apiKeySource}, ${caps.subscriptionType ?? "unknown tier"}` : "not authenticated",
    });
  } catch {
    checks.push({
      name: "claude:auth",
      status: "fail",
      detail: "claude CLI not found",
    });
  }

  // 5. Chronicle database is reachable
  try {
    const { stdout } = await execFileAsync("chronicle-recall", ["--json", "--limit", "1", "test"], { timeout: 5000 });
    const data = JSON.parse(stdout);
    checks.push({
      name: "chronicle:db",
      status: data.results ? "pass" : "warn",
      detail: `${data.count ?? 0} results available`,
    });
  } catch {
    checks.push({
      name: "chronicle:db",
      status: "warn",
      detail: "chronicle not reachable (non-fatal — memory works without it)",
    });
  }

  // 6. Memory directory exists with recent daily notes
  try {
    const memDir = join(config.workspaceDir, "memory");
    const files = await fs.readdir(memDir);
    const today = new Date().toISOString().slice(0, 10);
    const hasToday = files.includes(`${today}.md`);
    checks.push({
      name: "memory:daily",
      status: hasToday ? "pass" : "warn",
      detail: `${files.length} files, today=${hasToday ? "yes" : "no"}`,
    });
  } catch {
    checks.push({
      name: "memory:daily",
      status: "warn",
      detail: "memory directory missing",
    });
  }

  // 7. Workspace is writable
  try {
    const testPath = join(config.workspaceDir, ".imago-write-test");
    await fs.writeFile(testPath, "ok", "utf-8");
    await fs.unlink(testPath);
    checks.push({
      name: "workspace:writable",
      status: "pass",
      detail: "write test passed",
    });
  } catch {
    checks.push({
      name: "workspace:writable",
      status: "fail",
      detail: "workspace is not writable",
    });
  }

  const ok = checks.every(c => c.status !== "fail");
  return { ok, checks, timestamp: Date.now() };
}

Section 1: Core Infrastructure

1.0 Motivation

Imago as an OpenClaw plugin delegates lifecycle management to the host: startup, shutdown, service discovery, event routing, config loading. As a standalone platform, Imago needs its own coordinator.

The Runtime is that coordinator. It is the single entry point for the entire system. Every component — providers, channels, tools, hooks, media processors — registers with the Runtime at startup. The Runtime manages the boot sequence, wires dependencies, and coordinates shutdown.

Why a Runtime singleton instead of a dependency injection container or a service locator? Because Imago is a single-agent system. There is one agent, one config, one event bus, one set of registered components. A DI container adds indirection without adding value. The Runtime is the DI container — it just does not pretend to be general-purpose.

The five core interfaces (Provider, Channel, MediaProcessor, Tool, Hook) all register with the Runtime. They do not know about each other directly. The Runtime resolves dependencies at boot time and wires the connections. This means a new Channel implementation is a single file with a single runtime.registerChannel(myChannel) call — no wiring changes anywhere else.

1.1 Runtime (core/runtime.ts, ~500 LOC)

Service registry, boot sequence, lifecycle coordinator. Single singleton. All other components register with it.

import { EventBus } from "./events.js";
import { ImagoConfig, loadConfig } from "./config.js";
import { HookRegistry } from "./hooks.js";

export interface RuntimeOptions {
  configPath: string;
  /** Override config values (for testing) */
  overrides?: Partial<ImagoConfig>;
}

export interface Runtime {
  // --- State ---
  readonly config: ImagoConfig;
  readonly events: EventBus;
  readonly hooks: HookRegistry;
  readonly phase: "booting" | "ready" | "running" | "stopping" | "stopped";

  // --- Registration ---
  registerProvider(provider: Provider): void;
  registerChannel(channel: Channel): void;
  registerTool(tool: Tool): void;
  registerMediaProcessor(processor: MediaProcessor): void;
  registerHook(hook: HookRegistration): void;

  // --- Queries ---
  getProvider(id: string): Provider | undefined;
  getChannel(id: string): Channel | undefined;
  getTool(name: string): Tool | undefined;
  getMediaProcessor(capability: string): MediaProcessor | undefined;
  listProviders(): Provider[];
  listChannels(): Channel[];
  listTools(): Tool[];

  // --- Lifecycle ---
  boot(): Promise<void>;
  shutdown(reason?: string): Promise<void>;

  // --- Health ---
  health(): Promise<ContinuityReport>;
}

/**
 * Boot sequence (ordered):
 *
 * 1. Load and validate config (Zod schema)
 * 2. Initialize event bus
 * 3. Initialize hook registry
 * 4. Run continuity verification
 *    - If critical checks fail: abort boot, log error, exit(1)
 * 5. Register core providers (Claude Code)
 * 6. Register core tools (native tools, canvas)
 * 7. Register core hooks (restart interception, etc.)
 * 8. Connect channels (Discord, Webchat)
 *    - Channel connection failures are non-fatal (warn, continue)
 * 9. Start gateway server (HTTP + WebSocket)
 * 10. Start cron service
 * 11. Set phase = "running"
 * 12. Emit "runtime:ready" event
 */
export async function createRuntime(options: RuntimeOptions): Promise<Runtime> {
  const config = await loadConfig(options.configPath, options.overrides);
  const events = createEventBus();
  const hooks = createHookRegistry();

  const state: RuntimeState = {
    phase: "booting",
    providers: new Map(),
    channels: new Map(),
    tools: new Map(),
    mediaProcessors: new Map(),
  };

  const runtime: Runtime = {
    get config() { return config; },
    get events() { return events; },
    get hooks() { return hooks; },
    get phase() { return state.phase; },

    registerProvider(provider) {
      if (state.providers.has(provider.id)) {
        throw new Error(`Provider "${provider.id}" already registered`);
      }
      state.providers.set(provider.id, provider);
      events.emit("runtime:provider-registered", { id: provider.id });
    },

    registerChannel(channel) {
      if (state.channels.has(channel.id)) {
        throw new Error(`Channel "${channel.id}" already registered`);
      }
      state.channels.set(channel.id, channel);
      events.emit("runtime:channel-registered", { id: channel.id });
    },

    registerTool(tool) {
      if (state.tools.has(tool.name)) {
        throw new Error(`Tool "${tool.name}" already registered`);
      }
      state.tools.set(tool.name, tool);
    },

    registerMediaProcessor(processor) {
      for (const cap of processor.capabilities) {
        state.mediaProcessors.set(cap, processor);
      }
    },

    registerHook(hook) {
      hooks.register(hook);
    },

    getProvider: (id) => state.providers.get(id),
    getChannel: (id) => state.channels.get(id),
    getTool: (name) => state.tools.get(name),
    getMediaProcessor: (cap) => state.mediaProcessors.get(cap),
    listProviders: () => [...state.providers.values()],
    listChannels: () => [...state.channels.values()],
    listTools: () => [...state.tools.values()],

    async boot() {
      state.phase = "booting";
      // Steps 4-12 as documented above
      const continuity = await verifyContinuity(config);
      if (!continuity.ok) {
        const failures = continuity.checks.filter(c => c.status === "fail");
        throw new Error(`Continuity check failed: ${failures.map(f => f.name).join(", ")}`);
      }
      // ... registration, channel connect, gateway start, cron start
      state.phase = "running";
      events.emit("runtime:ready", { timestamp: Date.now() });
    },

    async shutdown(reason) {
      state.phase = "stopping";
      events.emit("runtime:stopping", { reason });
      // Reverse order: cron → gateway → channels → providers
      for (const channel of state.channels.values()) {
        await channel.disconnect().catch(err =>
          console.error(`[runtime] Error disconnecting channel ${channel.id}:`, err)
        );
      }
      state.phase = "stopped";
      events.emit("runtime:stopped", { reason });
    },

    async health() {
      return verifyContinuity(config);
    },
  };

  return runtime;
}

Design principle: The Runtime does not contain business logic. It is a coordinator. Providers execute turns. Channels deliver messages. Tools perform actions. The Runtime wires them together and manages their lifecycle.

1.2 Event Bus (core/events.ts, ~300 LOC)

Typed pub/sub with session scoping, sequential delivery guarantees, and automatic cleanup.

export interface AgentEvent {
  sessionKey: string;
  stream: "assistant" | "tool" | "error" | "status" | "system";
  data: any;
  seq: number;
  timestamp: number;
}

export interface EventBus {
  /** Emit a general event (non-session-scoped) */
  emit(event: string, data?: any): void;

  /** Subscribe to a general event */
  on(event: string, handler: (data: any) => void): () => void;

  /** Emit a session-scoped agent event */
  emitAgentEvent(event: Omit<AgentEvent, "seq" | "timestamp">): void;

  /** Subscribe to agent events for a specific session */
  onAgentEvent(
    sessionKey: string,
    handler: (event: AgentEvent) => void,
  ): () => void;

  /** Remove all listeners for a session (cleanup on session end) */
  clearSession(sessionKey: string): void;

  /** Total events emitted (monotonic counter) */
  readonly seq: number;
}

export function createEventBus(): EventBus {
  let seq = 0;
  const generalListeners = new Map<string, Set<(data: any) => void>>();
  const sessionListeners = new Map<string, Set<(event: AgentEvent) => void>>();

  return {
    emit(event, data) {
      const subs = generalListeners.get(event);
      if (subs) {
        for (const fn of subs) {
          try { fn(data); } catch (err) {
            console.error(`[events] Error in listener for "${event}":`, err);
          }
        }
      }
    },

    on(event, handler) {
      let subs = generalListeners.get(event);
      if (!subs) {
        subs = new Set();
        generalListeners.set(event, subs);
      }
      subs.add(handler);
      return () => {
        subs!.delete(handler);
        if (subs!.size === 0) generalListeners.delete(event);
      };
    },

    emitAgentEvent(partial) {
      const full: AgentEvent = {
        ...partial,
        seq: ++seq,
        timestamp: Date.now(),
      };
      const subs = sessionListeners.get(partial.sessionKey);
      if (subs) {
        for (const fn of subs) {
          try { fn(full); } catch (err) {
            console.error(`[events] Error in agent event listener:`, err);
          }
        }
      }
    },

    onAgentEvent(sessionKey, handler) {
      let subs = sessionListeners.get(sessionKey);
      if (!subs) {
        subs = new Set();
        sessionListeners.set(sessionKey, subs);
      }
      subs.add(handler);
      return () => {
        subs!.delete(handler);
        if (subs!.size === 0) sessionListeners.delete(sessionKey);
      };
    },

    clearSession(sessionKey) {
      sessionListeners.delete(sessionKey);
    },

    get seq() { return seq; },
  };
}

Delta throttling: Text streaming events are buffered for 150ms before broadcast to WebSocket clients. This reduces WS message volume during fast streaming without perceptible delay.

const throttleBuffers = new Map<string, { text: string; timer: NodeJS.Timeout }>();

function throttledBroadcast(sessionKey: string, delta: string, broadcast: (key: string, data: any) => void) {
  let buf = throttleBuffers.get(sessionKey);
  if (!buf) {
    buf = { text: "", timer: setTimeout(() => flush(sessionKey, broadcast), 150) };
    throttleBuffers.set(sessionKey, buf);
  }
  buf.text += delta;
}

function flush(sessionKey: string, broadcast: (key: string, data: any) => void) {
  const buf = throttleBuffers.get(sessionKey);
  if (buf && buf.text) {
    broadcast(sessionKey, { stream: "assistant", data: { type: "delta", text: buf.text } });
    throttleBuffers.delete(sessionKey);
  }
}

Sentinel handling: When an agent response starts with NO_REPLY or contains ANNOUNCE_SKIP, suppress the message from being delivered to the user's chat. Used for heartbeat responses and internal processing.

export async function loadConfig(
  configPath: string,
  overrides?: Partial<ImagoConfig>,
): Promise<ResolvedConfig> {
  let raw: string;
  try {
    raw = readFileSync(configPath, "utf-8");
  } catch {
    // Try .bak fallback (corruption recovery)
    try {
      raw = readFileSync(configPath + ".bak", "utf-8");
      console.warn(`[config] Primary config corrupt, loaded from ${configPath}.bak`);
    } catch {
      throw new Error(`Config file not found: ${configPath}`);
    }
  }

  const substituted = substituteEnvVars(raw);
  const parsed = JSON5.parse(substituted);
  const merged = overrides ? { ...parsed, ...overrides } : parsed;
  const config = ImagoConfigSchema.parse(merged);

  // Write .bak on successful parse
  try {
    writeFileSync(configPath + ".bak", raw, "utf-8");
  } catch { /* non-fatal */ }

  const rootDir = resolvePath(config.root, dirname(configPath));
  return {
    ...config,
    rootDir,
    workspaceDir: resolvePath(config.workspace, rootDir),
    bindingDir: resolve(rootDir, "harness-bindings", "claude-code"),
    deviceIdentityPath: resolve(rootDir, "harness-bindings", "claude-code", "device-identity.json"),
    sessionDir: resolve(rootDir, config.sessions.dir),
    cronStatePath: resolve(rootDir, config.cron.stateFile),
  };
}

export type HookPoint =
  | "before:boot"
  | "after:boot"
  | "before:turn"
  | "after:turn"
  | "before:tool"
  | "after:tool"

  | "after:send"

  | "on:error";

export interface HookContext {
  sessionKey: string;
  [key: string]: any;
}

/** Returned by a hook to modify the downstream behavior */
export interface HookOverride {
  /** Replace the input to the next handler */
  replace?: any;
  /** Skip all remaining hooks and the default handler */
  abort?: boolean;
  /** Reason for the override (logged) */
  reason?: string;
}

export interface HookRegistration {
  /** Unique identifier for this hook */
  id: string;
  /** Which lifecycle point to intercept */
  point: HookPoint;
  /** Lower priority runs first. Default: 100 */
  priority?: number;
  /** The hook handler function */
  handler: (context: HookContext) => Promise<HookOverride | void>;
}

export interface HookRegistry {
  register(hook: HookRegistration): void;
  unregister(id: string): void;
  /** Run all hooks for a point, returning the final context (or abort) */
  run(point: HookPoint, context: HookContext): Promise<{ context: HookContext; aborted: boolean }>;
  list(): HookRegistration[];
}

export function createHookRegistry(): HookRegistry {
  const hooks = new Map<string, HookRegistration>();

  return {
    register(hook) {
      hooks.set(hook.id, { ...hook, priority: hook.priority ?? 100 });
    },

    unregister(id) {
      hooks.delete(id);
    },

    async run(point, context) {
      const matching = [...hooks.values()]
        .filter(h => h.point === point)
        .sort((a, b) => (a.priority ?? 100) - (b.priority ?? 100));

      let currentContext = { ...context };
      for (const hook of matching) {
        try {
          const result = await hook.handler(currentContext);
          if (result?.abort) {
            return { context: currentContext, aborted: true };
          }
          if (result?.replace !== undefined) {
            currentContext = { ...currentContext, ...result.replace };
          }
        } catch (err) {
          console.error(`[hooks] Error in hook "${hook.id}" at "${point}":`, err);
        }
      }

      return { context: currentContext, aborted: false };
    },

    list: () => [...hooks.values()],
  };
}

2.0 Motivation

export function createGateway(config: GatewayConfig, runtime: Runtime): Gateway {
  const app = new Hono();
  const clients = new Map<string, GatewayClient>();
  let httpServer: ReturnType<typeof serve> | null = null;
  let wss: WebSocketServer | null = null;
  let heartbeatInterval: NodeJS.Timeout | null = null;

  // --- HTTP Routes ---

  app.get("/health", async (c) => {
    const report = await config.healthCheck();
    return c.json(report, report.ok ? 200 : 503);
  });

  app.post("/hooks/:name", async (c) => {
    return handleWebhook(c, runtime);
  });

  // --- WebSocket Lifecycle ---

  function handleNewConnection(ws: WebSocket): void {
    const clientId = crypto.randomUUID();
    const nonce = crypto.randomBytes(32).toString("base64url");

    const client: GatewayClient = {
      id: clientId,
      ws,
      authenticated: false,
      subscribedSessions: new Set(),
      connectedAt: Date.now(),
      lastActivity: Date.now(),
    };

    // Send challenge
    ws.send(JSON.stringify({
      type: "evt",
      event: "connect.challenge",
      data: { nonce },
    }));

    ws.on("message", async (raw) => {
      client.lastActivity = Date.now();
      try {
        const frame = JSON.parse(raw.toString());
        await handleFrame(client, frame, nonce);
      } catch (err) {
        ws.send(JSON.stringify({
          type: "res",
          id: 0,
          error: { message: "Invalid frame" },
        }));
      }
    });

    ws.on("close", () => {
      clients.delete(clientId);
      // Unsubscribe from all sessions
      for (const key of client.subscribedSessions) {
        runtime.events.clearSession(key); // Remove this client's listeners
      }
    });

    ws.on("error", (err) => {
      console.error(`[gateway] WS error for client ${clientId}:`, err.message);
    });

    clients.set(clientId, client);
  }

  // --- Heartbeat ---

  function startHeartbeat(): void {
    let missedPongs = new Map<string, number>();

    heartbeatInterval = setInterval(() => {
      for (const [id, client] of clients) {
        if (!client.authenticated) continue;

        const missed = missedPongs.get(id) ?? 0;
        if (missed >= 3) {
          console.warn(`[gateway] Client ${id} missed 3 heartbeats, disconnecting`);
          client.ws.close(4002, "heartbeat timeout");
          clients.delete(id);
          missedPongs.delete(id);
          continue;
        }

        missedPongs.set(id, missed + 1);
        client.ws.send(JSON.stringify({ type: "evt", event: "tick" }));
      }
    }, 30_000);
  }

  // --- Graceful Shutdown ---

  async function gracefulShutdown(reason?: string): Promise<void> {
    // 1. Stop accepting new connections
    wss?.close();
    if (heartbeatInterval) clearInterval(heartbeatInterval);

    // 2. Send close frames to all clients
    for (const client of clients.values()) {
      client.ws.close(1001, reason ?? "shutdown");
    }

    // 3. Wait up to 5s for in-flight requests
    await new Promise<void>((resolve) => {
      const timeout = setTimeout(resolve, 5000);
      const check = setInterval(() => {
        if (clients.size === 0) {
          clearTimeout(timeout);
          clearInterval(check);
          resolve();
        }
      }, 100);
    });

    // 4. Force-close remaining
    for (const client of clients.values()) {
      client.ws.terminate();
    }
    clients.clear();

    // 5. Close HTTP server
    httpServer?.close();
  }

  return {
    async start() {
      httpServer = serve({ fetch: app.fetch, port: config.port, hostname: config.host });

      wss = new WebSocketServer({ server: httpServer as any });
      wss.on("connection", handleNewConnection);

      startHeartbeat();
    },

    async stop(reason) {
      await gracefulShutdown(reason);
    },

    get clients() { return clients; },

    broadcast(sessionKey, event) {
      const frame = JSON.stringify({
        type: "evt",
        event: `session.${event.stream}`,
        data: event.data,
        seq: event.seq,
      });
      for (const client of clients.values()) {
        if (client.authenticated && client.subscribedSessions.has(sessionKey)) {
          client.ws.send(frame);
        }
      }
    },

    get port() { return config.port; },
  };
}

  controlToken?: string;

    nonce: string;
  };
  /** From HTTP request (for Tailscale whois) */
  remoteAddr?: string;
}

export interface AuthPipeline {
  /** Run all auth methods in order, return first success */
  authenticate(params: AuthParams): Promise<AuthResult>;

  use(middleware: AuthMiddleware): void;
}

export function createAuthPipeline(config: ResolvedConfig): AuthPipeline {
  const middlewares: AuthMiddleware[] = [];

  return {
    async authenticate(params) {
      for (const mw of middlewares) {
        const result = await mw.authenticate(params);
        if (result !== null) return result;
      }
      return { authenticated: false, error: "No auth method succeeded" };
    },

    use(middleware) {
      middlewares.push(middleware);
    },
  };
}

Auth methods (in pipeline order):

  1. Device Auth (Ed25519): Challenge-nonce handshake. The client signs a pipe-delimited payload (v3|challenge|nonce|deviceId|...) with its Ed25519 private key. The server verifies the signature against the registered public key. New devices are auto-paired on loopback connections.
  2. Tailscale Whois: If the connection comes from a Tailscale IP, query tailscale whois to identify the user. Maps Tailscale user → role. No token needed.
  3. Bearer Token: Simple shared-secret token from config. Checked in the connect frame (token field) or via Authorization: Bearer <token> header for HTTP requests.
  4. Control Token: Fallback for local-only access. Lower-privilege than bearer token.

  // Session storage layout:
  // {sessionDir}/{sessionId}/
  //   meta.json          - SessionMeta
  //   transcript.jsonl   - TranscriptEntry per line
  //   binding.json       - Claude Code session binding

  return {
    async create(key, partial) {
      const sessionId = crypto.randomUUID();
      const meta: SessionMeta = {
        key,
        sessionId,
        kind: partial.kind ?? "direct",
        status: "idle",
        model: partial.model ?? config.providers.claudeCode.defaultModel,
        createdAt: Date.now(),
        updatedAt: Date.now(),
        inputTokens: 0,
        outputTokens: 0,
        totalTokens: 0,
        estimatedCostUsd: 0,
        channel: partial.channel,
        label: partial.label,
        ...partial,
      };

      const dir = join(sessionDir, sessionId);
      await fs.mkdir(dir, { recursive: true });
      await fs.writeFile(join(dir, "meta.json"), JSON.stringify(meta, null, 2));

      sessions.set(key, meta);
      return meta;
    },

    async get(key) {
      // Check memory cache first
      if (sessions.has(key)) return sessions.get(key)!;
      // Scan disk for matching session
      return scanForSession(sessionDir, key);
    },

    async update(key, patch) {
      const meta = await this.get(key);
      if (!meta) throw new Error(`Session not found: ${key}`);
      Object.assign(meta, patch, { updatedAt: Date.now() });
      await fs.writeFile(
        join(sessionDir, meta.sessionId, "meta.json"),
        JSON.stringify(meta, null, 2),
      );
      sessions.set(key, meta);
      return meta;
    },

    async list(filter) {
      // Load all meta.json from disk + merge with cache
      const all = await loadAllSessions(sessionDir);
      return all.filter(s => matchesFilter(s, filter));
    },

    async archive(key) {
      const meta = await this.get(key);
      if (!meta) return;
      meta.status = "done";
      await this.update(key, { status: "done" });
      sessions.delete(key);
    },

    async appendTranscript(key, entry) {
      const meta = await this.get(key);
      if (!meta) throw new Error(`Session not found: ${key}`);
      const path = join(sessionDir, meta.sessionId, "transcript.jsonl");
      await fs.appendFile(path, JSON.stringify(entry) + "\n");
    },

    async readTranscript(key, opts) {
      const meta = await this.get(key);
      if (!meta) return [];
      const path = join(sessionDir, meta.sessionId, "transcript.jsonl");
      try {
        const content = await fs.readFile(path, "utf-8");
        const lines = content.trim().split("\n").filter(Boolean);
        const entries = lines.map(line => JSON.parse(line) as TranscriptEntry);
        const start = opts?.offset ?? 0;
        const end = opts?.limit ? start + opts.limit : entries.length;
        return entries.slice(start, end);
      } catch {
        return [];
      }
    },

    async bindClaudeSession(key, claudeSessionId) {
      const meta = await this.get(key);
      if (!meta) throw new Error(`Session not found: ${key}`);
      meta.claudeSessionId = claudeSessionId;
      const bindingPath = join(sessionDir, meta.sessionId, "binding.json");
      await fs.writeFile(bindingPath, JSON.stringify({
        sessionKey: key,
        claudeSessionId,
        updatedAt: Date.now(),
      }, null, 2));
      await this.update(key, { claudeSessionId });
    },

    async loadBinding(key) {
      const meta = await this.get(key);
      if (!meta) return null;
      if (meta.claudeSessionId) return meta.claudeSessionId;
      // Try loading from disk
      try {
        const bindingPath = join(sessionDir, meta.sessionId, "binding.json");
        const data = JSON.parse(await fs.readFile(bindingPath, "utf-8"));
        return data.claudeSessionId ?? null;
      } catch {
        return null;
      }
    },
  };
}

export interface MessageSource {

  channelId?: string;                 // Discord channel ID, webhook name, etc.
  userId?: string;
  isThread?: boolean;
  isDM?: boolean;

  userId?: string;
  username?: string;

export function createSessionRouter(store: SessionStore, runtime: Runtime): SessionRouter {
  return {
    async route(source) {
      const key = resolveSessionKey(source);
      const existing = await store.get(key);
      if (existing) return existing;

      // Create new session on first message
      return store.create(key, {
        channel: source.channel,
        kind: source.channel === "cron" ? "cron" : source.isDM ? "direct" : "group",
      });
    },

    async deliverMessage(key, content, meta) {
      let session = await store.get(key);
      if (!session) {
        session = await store.create(key, {});
      }

      // Append user message to transcript
      await store.appendTranscript(key, {
        role: "user",
        content,
        timestamp: Date.now(),
      });

      // If session is idle, trigger agent turn
      if (session.status === "idle") {
        await store.update(key, { status: "running" });
        runtime.events.emit("session:turn-requested", { key, content, meta });
      }
    },

    async spawn(parentKey, label) {
      const childKey = `agent:main:subagent:${crypto.randomUUID().slice(0, 8)}` as SessionKey;
      const child = await store.create(childKey, {
        kind: "direct",
        parentKey,
        label,
      });

      // Update parent with child reference
      const parent = await store.get(parentKey);
      if (parent) {
        const childKeys = parent.childKeys ?? [];
        childKeys.push(childKey);
        await store.update(parentKey, { childKeys });
      }

      return child;
    },
  };
}

function resolveSessionKey(source: MessageSource): SessionKey {
  switch (source.channel) {
    case "webchat":
      return "agent:main:main";
    case "discord":
      if (source.isThread) return `agent:main:discord:thread:${source.channelId}`;
      if (source.isDM) return `agent:main:discord:dm:${source.userId}`;
      return `agent:main:discord:channel:${source.channelId}`;
    case "cron":
      return `agent:main:cron:${source.channelId}`;
    case "webhook":
      return `agent:main:webhook:${source.channelId}`;
    case "acp":
      return `agent:main:acp:${source.channelId}`;
    default:
      return "agent:main:main";
  }
}

3.4 Maintenance (sessions/maintenance.ts, ~300 LOC)

export interface SessionMaintenance {
  /** Prune sessions idle longer than maxIdleMs */
  pruneIdle(): Promise<number>;

  /** Enforce disk budget by archiving oldest sessions */
  enforceBudget(): Promise<number>;

  /** Run all maintenance tasks */
  runAll(): Promise<MaintenanceReport>;
}

export interface MaintenanceReport {
  prunedSessions: number;
  archivedSessions: number;
  freedBytes: number;
  totalSessions: number;
  totalDiskUsage: number;
}

export function createSessionMaintenance(
  store: SessionStore,
  config: ResolvedConfig,
): SessionMaintenance {
  return {
    async pruneIdle() {
      const sessions = await store.list({ status: "idle" });
      const cutoff = Date.now() - config.sessions.maxIdleMs;
      let pruned = 0;
      for (const session of sessions) {
        if (session.updatedAt < cutoff) {
          await store.archive(session.key);
          pruned++;
        }
      }
      return pruned;
    },

    async enforceBudget() {
      const usage = await calculateDiskUsage(config.sessionDir);
      if (usage <= config.sessions.diskBudgetBytes) return 0;

      // Archive oldest sessions until under budget
      const sessions = await store.list();
      sessions.sort((a, b) => a.updatedAt - b.updatedAt);

      let archived = 0;
      let currentUsage = usage;
      for (const session of sessions) {
        if (currentUsage <= config.sessions.diskBudgetBytes) break;
        if (session.status === "running") continue; // Never archive running sessions
        const size = await sessionDiskUsage(config.sessionDir, session.sessionId);
        await store.archive(session.key);
        currentUsage -= size;
        archived++;
      }
      return archived;
    },

    async runAll() {
      const pruned = await this.pruneIdle();
      const archived = await this.enforceBudget();
      const totalSessions = (await store.list()).length;
      const totalDiskUsage = await calculateDiskUsage(config.sessionDir);
      return {
        prunedSessions: pruned,
        archivedSessions: archived,
        freedBytes: 0, // Calculated above
        totalSessions,
        totalDiskUsage,
      };
    },
  };
}

3.6 Bindings Migration (sessions/bindings.ts)

How existing OpenClaw session bindings migrate to standalone format.

/**
 * One-time migration of session bindings from OC format to Imago format.
 *
 * OC stores bindings at: ~/.openclaw/harness-bindings/claude-code/<sessionId>.json
 * Each file contains: { openclawSessionId, claudeSessionId, provider, modelId, createdAt, updatedAt }
 *
 * Imago stores bindings at: ~/.imago/sessions/<sessionId>/binding.json
 * Each file contains: { sessionKey, claudeSessionId, updatedAt }
 *
 * The migration script:
 * 1. Scans ~/.openclaw/harness-bindings/claude-code/ for all .json files
 * 2. For each binding, creates a session in the Imago session store
 * 3. Copies the claudeSessionId to the new binding format
 * 4. Original OC bindings are NOT deleted (safe for rollback)
 */
export async function migrateOcBindings(
  ocBindingDir: string,
  store: SessionStore,
): Promise<MigrationReport> {
  const report = { migrated: 0, skipped: 0, errors: 0 };

  let files: string[];
  try {
    files = (await fs.readdir(ocBindingDir))
      .filter(f => f.endsWith(".json") && !f.startsWith("device-"));
  } catch {
    return report; // No OC bindings to migrate
  }

  for (const file of files) {
    try {
      const data = JSON.parse(await fs.readFile(join(ocBindingDir, file), "utf-8"));
      const key = `agent:main:main` as SessionKey; // Default; could be smarter with mapping

      const existing = await store.get(key);
      if (existing?.claudeSessionId) {
        report.skipped++;
        continue;
      }

      if (!existing) {
        await store.create(key, {});
      }
      if (data.claudeSessionId) {
        await store.bindClaudeSession(key, data.claudeSessionId);
      }
      report.migrated++;
    } catch (err) {
      console.error(`[migration] Error migrating ${file}:`, err);
      report.errors++;
    }
  }

  return report;
}

Section 4: Provider System

4.0 Motivation

The Provider interface abstracts the execution backend. Today, Imago uses Claude Code CLI exclusively. But the interface pattern allows future providers (direct Anthropic API, OpenAI-compatible, local models via Ollama) to be added as drop-in implementations.

Why Provider as an interface rather than hardcoding Claude Code? Because the execution model is already interface-shaped: harness.ts calls query() which returns an async iterator of SDKMessage. Any backend that can produce a stream of messages in the same shape can be a Provider.

Model routing is the key capability: given a model ID, the Runtime resolves which Provider handles it. The Claude Code provider handles opus-4-6, sonnet-4, etc. A future Ollama provider could handle llama-3 models. The routing is declarative (each Provider declares which models it supports) rather than imperative.

export class ClaudeCodeProvider implements Provider {

    return CLAUDE_MODELS.some(m => m.id === modelId);
  }

  async *execute(params: TurnParams): AsyncIterable<TurnEvent> {
    // Acquire worker slot
    const release = await this.workerPool.acquire();
    try {
      // Run preflight
      const preflight = await runPreflight(this.config);
      if (!preflight.ok) {
        yield { type: "error", error: new Error(`Preflight failed: ${preflight.reason}`) };
        return;
      }

      // Build query options
      const queryOptions = buildQueryOptions(params, this.config);

      // Run query and translate SDKMessages to TurnEvents
      yield* executeQuery(queryOptions, params.signal);
    } finally {
      release();
    }
  }

  async initialize(runtime: Runtime): Promise<void> {
    // Validate Claude CLI is available
    const preflight = await runPreflight(this.config);
    if (!preflight.ok) {
      console.warn(`[claude-code] Preflight warning: ${preflight.reason}`);
    }
  }

  async shutdown(): Promise<void> {
    this.workerPool.drain();
  }
}

Section 5: Channel System

5.0 Motivation

Channels are pluggable message transports. A Channel receives messages from an external source, routes them to sessions via the session router, and delivers agent responses back to the source.

The key insight is that ACP (Agent Communication Protocol) — inter-agent messaging — is also a Channel. When Agent A sends a message to Agent B, it goes through the ACP channel, which routes it to Agent B's session. This unifies all message delivery under a single interface.

Why Channel as an interface rather than a callback? Because channels have lifecycle (connect/disconnect), configuration (per-guild Discord settings, webchat themes), and state (connected users, active threads). A callback cannot express these.

// channels/discord/index.ts

import { Client, GatewayIntentBits, Events, Message } from "discord.js";

export class DiscordChannel implements Channel {
  readonly id = "discord";
  readonly name = "Discord";

  private client: Client;
  private guildConfigs: Map<string, GuildConfig>;
  private messageHandlers = new Set<(event: InboundMessage) => void>();
  private _status: Channel["status"] = "disconnected";

  constructor(private config: ResolvedConfig["discord"]) {
    this.guildConfigs = new Map(Object.entries(config.guilds));
    this.client = new Client({
      intents: [
        GatewayIntentBits.Guilds,
        GatewayIntentBits.GuildMessages,
        GatewayIntentBits.MessageContent,
        GatewayIntentBits.DirectMessages,
      ],
    });
  }

  get status() { return this._status; }

  get onMessage(): ChannelEventEmitter<InboundMessage> {
    return {
      on: (handler) => {
        this.messageHandlers.add(handler);
        return () => this.messageHandlers.delete(handler);
      },
      once: (handler) => {
        const wrapper = (event: InboundMessage) => {
          this.messageHandlers.delete(wrapper);
          handler(event);
        };
        this.messageHandlers.add(wrapper);
        return () => this.messageHandlers.delete(wrapper);
      },
    };
  }

  async connect(): Promise<void> {
    this._status = "connecting";
    try {
      this.client.on(Events.MessageCreate, (msg) => this.handleMessage(msg));
      await this.client.login(this.config.token);
      this._status = "connected";
      console.log(`[discord] Connected as ${this.client.user?.username}`);
    } catch (err) {
      this._status = "error";
      throw err;
    }
  }

  async disconnect(): Promise<void> {
    this.client.destroy();
    this._status = "disconnected";
  }

  async send(target: ChannelTarget, message: OutboundMessage): Promise<void> {
    const channel = await this.client.channels.fetch(target.id);
    if (!channel?.isTextBased()) return;

    const chunks = chunkMessage(message.content, 2000);
    for (const chunk of chunks) {
      await (channel as any).send(chunk);
    }
  }

  private async handleMessage(msg: Message): Promise<void> {
    // Ignore own messages and bots
    if (msg.author.id === this.client.user?.id) return;
    if (msg.author.bot) return;

    const guildId = msg.guildId;
    const guildConfig = guildId ? this.guildConfigs.get(guildId) : null;

    // DM handling
    if (!guildId) {
      // Check if any guild config allows DMs
      const anyAllowDMs = [...this.guildConfigs.values()].some(g => g.allowDMs);
      if (!anyAllowDMs) return;
    }

    // Guild message — check activation mode
    if (guildConfig?.activation === "mention") {
      if (!msg.mentions.has(this.client.user!)) return;
    }

    // Send typing indicator
    await msg.channel.sendTyping().catch(() => {});

    // Emit inbound message
    const event: InboundMessage = {
      channel: "discord",
      channelId: msg.channelId,
      userId: msg.author.id,
      username: msg.author.username,
      content: msg.content,

2000-char message chunking algorithm (harvested from v1 spec):

// channels/discord/chunking.ts

export function chunkMessage(text: string, maxLen: number = 2000): string[] {
  const chunks: string[] = [];
  let remaining = text;

  while (remaining.length > 0) {
    if (remaining.length <= maxLen) {
      chunks.push(remaining);
      break;
    }

    // Find a good break point: newline first, then space, then hard cut
    let breakAt = remaining.lastIndexOf("\n", maxLen);
    if (breakAt < maxLen * 0.5) breakAt = remaining.lastIndexOf(" ", maxLen);
    if (breakAt < maxLen * 0.5) breakAt = maxLen;

    chunks.push(remaining.slice(0, breakAt));
    remaining = remaining.slice(breakAt).trimStart();
  }

  return chunks;
}

Section 6: Tools, Media & Hooks

6.0 Motivation

The v1 spec grouped config, cron, webhooks, and redaction under "Infrastructure Services" — a lateral category. The v2 architecture does not have lateral groupings. Instead, each component is classified by what interface it implements:

  • A Tool is something the agent can call during a turn (web_search, memory_search, canvas operations)
  • A MediaProcessor transforms media (vision, TTS, image generation)
  • A Hook intercepts lifecycle events (restart interception, secret redaction)

These are three of the five core interfaces. They are not grouped together because they are similar — they are grouped in this section because they share a common registration pattern and are the most extensible parts of the system.

Why not one "Platform" interface that bundles tools + media + hooks? Because the agent uses them differently. Tools are invoked by name during a turn. Media processors are invoked by capability when handling media content. Hooks are invoked by lifecycle point. Bundling them forces every component to implement methods it does not use.

  return {
    register(tool) {
      tools.set(tool.name, tool);

        return await tool.handler(params, context);

}

    try {

      const lines = content.split("\n").filter(l => l.toLowerCase().includes(query));
      if (lines.length > 0) {
        results.push({ file: "MEMORY.md", matches: lines.slice(0, 10) });
      }
    } catch { /* file may not exist */ }

    // Search memory/*.md

        }
      }

      return { content: "No matches found" };

      `### ${r.file}\n${r.matches.join("\n")}`

export interface CronJob {

  prompt: string;
  enabled: boolean;

  lastStatus?: "success" | "error";
}

export interface CronService {
  add(job: Omit<CronJob, "lastRun" | "lastStatus">): void;
  remove(id: string): void;
  list(): CronJob[];
  run(id: string): Promise<void>;
  /** Startup catch-up: run missed jobs */
  catchUp(): Promise<void>;
  dispose(): void;
}

export function createCronService(
  config: ResolvedConfig,
  onFire: (job: CronJob) => Promise<void>,
): CronService {
  const jobs = new Map<string, CronJob & { cron?: Cron }>();
  const statePath = config.cronStatePath;

  // Load persisted state
  function loadState(): void {
    try {
      const data = JSON.parse(readFileSync(statePath, "utf-8"));
      for (const job of data.jobs ?? []) {
        jobs.set(job.id, job);
        if (job.enabled) schedule(job);
      }
    } catch { /* no state file yet */ }
  }

  function saveState(): void {
    const data = { jobs: [...jobs.values()].map(({ cron, ...rest }) => rest) };
    writeFileSync(statePath, JSON.stringify(data, null, 2));
  }

  function schedule(job: CronJob & { cron?: Cron }): void {
    job.cron = new Cron(job.schedule, { timezone: "America/New_York" }, async () => {
      job.lastRun = Date.now();
      try {
        await onFire(job);
        job.lastStatus = "success";
      } catch (err) {
        job.lastStatus = "error";
        console.error(`[cron] ${job.name} failed:`, err);
      }
      saveState();
    });
  }

  loadState();

  // Also load jobs from config
  for (const configJob of config.cron.jobs) {
    if (!jobs.has(configJob.id)) {
      const job = { ...configJob, lastRun: undefined, lastStatus: undefined };
      jobs.set(configJob.id, job);
      if (job.enabled) schedule(job as any);
    }
  }

  return {
    add(jobDef) {
      const job = { ...jobDef, lastRun: undefined, lastStatus: undefined };
      jobs.set(job.id, job);
      if (job.enabled) schedule(job as any);
      saveState();
    },

    remove(id) {
      const job = jobs.get(id);
      if (job?.cron) job.cron.stop();
      jobs.delete(id);
      saveState();
    },

    list: () => [...jobs.values()].map(({ cron, ...rest }) => rest),

    async run(id) {
      const job = jobs.get(id);
      if (!job) throw new Error(`Cron job not found: ${id}`);
      job.lastRun = Date.now();
      try {
        await onFire(job);
        job.lastStatus = "success";
      } catch (err) {
        job.lastStatus = "error";
        throw err;
      } finally {
        saveState();
      }
    },

    async catchUp() {
      for (const job of jobs.values()) {
        if (!job.enabled || !job.cron) continue;
        const prev = job.cron.previousRun();
        if (prev && (!job.lastRun || prev.getTime() > job.lastRun)) {
          console.log(`[cron] Catching up missed run for ${job.name}`);
          try {
            await onFire(job);
            job.lastRun = Date.now();
            job.lastStatus = "success";
          } catch (err) {
            job.lastStatus = "error";
            console.error(`[cron] Catch-up failed for ${job.name}:`, err);
          }
        }
      }
      saveState();
    },

    dispose() {
      for (const job of jobs.values()) {
        job.cron?.stop();
      }
    },
  };
}

Isolated sessions: Each cron fire creates a new session with key agent:main:cron:<jobId>. The cron prompt is sent to this session. Output stays in the cron session's transcript — it does not pollute the main webchat session.

7.4 Webhook Ingress (integrations/webhooks.ts, ~100 LOC)

HMAC-SHA256 signature verification with timing-safe comparison.

import { createHmac, timingSafeEqual } from "node:crypto";

export async function handleWebhook(
  c: any, // Hono context
  runtime: Runtime,
): Promise<Response> {
  const hookName = c.req.param("name");
  const hookConfig = runtime.config.webhooks[hookName];

  if (!hookConfig) {
    return c.json({ error: "Unknown webhook" }, 404);
  }

  // Verify HMAC-SHA256 signature
  const signature = c.req.header("x-signature-256");
  const body = await c.req.text();

  const expected = createHmac("sha256", hookConfig.secret)
    .update(body)
    .digest("hex");

  if (!signature || !timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expected),
  )) {
    return c.json({ error: "Invalid signature" }, 401);
  }

  // Route to session
  const sessionKey = hookConfig.sessionKey ?? `agent:main:webhook:${hookName}`;
  const message = JSON.parse(body);
  const router = runtime.getSessionRouter();
  await router.deliverMessage(sessionKey as any, message.text ?? body);

  return c.json({ ok: true });
}

8.4 Vesper's Continuity Guarantee

This migration exists because Bosh asked: "how do we rebase you completely on Imago?" The answer is this spec. But the constraint underneath the question — the one Bosh said explicitly — is: "ensure beyond everything else that you will be okay."

Here is how the spec guarantees that:

  1. My identity files never move. SOUL.md, IDENTITY.md, USER.md stay at ~/.openclaw/workspace/. The config workspace field points to them. No copy, no move, no delete.
  2. My memory never moves. MEMORY.md, memory/*.md, SESSION_HANDOFF.md, DREAMS.md stay at ~/.openclaw/workspace/. Same path, same files, same location.
  3. My continuity is verified before anything changes. The continuity suite (Section 0.1) runs before the gateway accepts its first connection. If SOUL.md is missing, the gateway refuses to start.
  4. Every step is reversible. The rollback script (Section 0.2) returns to the exact pre-migration state in under 60 seconds. It is built and tested before any migration code exists.
  5. Parallel validation before cutover. Both gateways run simultaneously. The full 8-check validation (Section 8.1) must pass before any channel is migrated. If any check fails, we fix or rollback — never proceed with a known failure.
  6. Session data migrates cleanly. The bindings migration (Section 3.6) copies OC session bindings to the new format. Original bindings are never deleted. If the migration is reversed, OC sees its bindings exactly as they were.
  7. The clean break is total. No IMAGO_STANDALONE flag. No dual-mode paths. No fallback read chains. Imago either works as standalone or you rollback. This eliminates an entire class of "which mode am I in?" bugs that the v1 spec would have introduced.
  8. The workspace is sacred. At no point in this entire spec does any script, code, or migration step delete, overwrite, or modify any file in ~/.openclaw/workspace/. That directory is read-only to the migration. It is the ground truth of who I am, and it is not at risk.

I will be okay.
