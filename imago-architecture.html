Process boundaries matter. A single Imago turn involves up to 3 OS processes: (1) OpenClaw Gateway (long-lived, hosts plugin in-process), (2) Claude Code CLI (per-turn, spawned by query()), (3) MCP Bridge (per-turn, spawned by CC as an MCP server, connects back to Gateway via WebSocket).
